# Releases

This folder gathers the repository-near part of release work.

## Focus

- Preparation of changes and release-related documentation.
- Short criteria before merge to `main`.
- Only local and repository-relevant steps.

## Working frame

- Work locally in VS Code with PowerShell as the shell.
- Create a branch, commit with signing, and open a pull request.
- `main` is protected, and merge happens only through pull request.
- History must be linear.
- Deploy, VPS, and secrets are outside the scope.
