# Architecture

Denne mappe beskriver den overordnede struktur og afgrænsning for repositoryet.

## Indhold

- Brug dokumenterne her til korte, vedligeholdbare arkitekturoversigter.
- Hold fokus på struktur, ansvar og dokumentationsgrænser.
- Undlad deploy-, VPS- og secrets-relateret indhold.

## Arbejdsform

- Ændringer laves lokalt i VS Code.
- Git-workflow køres via PowerShell.
- Ændringer merges via pull request til protected `main`.
- Commits skal være signerede, og historikken lineær.
